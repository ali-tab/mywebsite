SmartPhind

SmartPhind uses a web scraper that visits popular smartphone manufacturing websites and fetches their latest products for convenient display on a GUI.
At the user’s request, program will visit popular Canadian mobile service provider websites and display available offers in a list.
Written in Python and uses Tkinter (Python’s standard GUI interface), headless browsing for interacting with web pages (clicking JavaScript buttons), and a web scraper for parsing through HTML code to find desired information.

It also uses a headless browser (“invisible” web browser) to interact with websites that have information otherwise hidden without interaction from website visitors (“Show more” buttons, etc.), before passing the HTML code to the web scraper.

Pang

This project is a web browser game that I am currently working on with a colleague. It builds off gameplay of the classic computer game, Pong, where two opponents have rectangular paddles, and they bounce a ball back and forth.

I have been developing the client and website layout, with CSS and HTML. The game itself is written in JavaScript, using the popular JavaScript API Three.js, for displaying animated 3D graphics.

So far, my client can connect to the server, and multiplayer is currently in testing mode – but it works! Two clients can connect to the server and play a game with each other, bouncing the ball back and forth.

The game takes keyboard inputs to play, which move the user’s paddle up and down.
Currently, big challenges we are facing are in regards with the server-side physics for the ball, and hosting multiple game sessions at the same time.

CrimeCheck

CrimeCheck is as program written in Java that uses weighing and pathfinding algorithms to determine the safest route from one tourist attraction to another in New York City, using the New York City Police’s 2019 recorded crime dataset. It uses Dijkstra’s Algorithm, a very well-known pathfinding algorithm, to find paths in a connected graph with 50+ nodes. Each node represents a tourist attraction.

The dataset used came from a .cvs file, and contained information like the latitude and longitude of each crime, the type of crime, the date of occurance, and classification of crime. This file was quite large, having over 460,000 entries. The program parses through the dataset, and stores this information for every single crime.
Then, the program weighes each node based on how many crimes occurred in its radius, and the classifications of the crime. Certain tourist locations are regarded as being more safe or dangerous compared to others, and that dictates the weight of each node for the pathfinding algorithm to use.

The GUI was developed using Java AWT (Abstract Window Toolkit).
Overall, the biggest challenge was how to weigh each node and the reasoning behind it. I used a point system, assigning bigger numbers to crimes that should be of more concern. If the crime occurred a few years ago, it would be worth less than a crime that occurred recently. Felonies added 12 points, misdeamanors added 6 points, and violations added 3 points.
